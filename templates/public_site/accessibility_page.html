Accessibility Standards Compliance

{% if page.standards_text %} {{ page.standards_text|richtext }} {% else %}

Our website has been designed and developed to meet Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards. We have achieved comprehensive compliance across all key accessibility areas:

  • Perceivable: All content is presentable to users in ways they can perceive
  • Operable: All interface components are operable by all users
  • Understandable: Information and UI operation are understandable
  • Robust: Content works with assistive technologies

Compliance Status: This website is fully compliant with WCAG 2.1 Level AA standards as of {{ "now"|date:"F Y" }}.

Accessibility Features

{% if page.features_text %} {{ page.features_text|richtext }} {% else %}

Navigation and Interaction

  • Skip navigation links for keyboard users
  • Consistent navigation structure across all pages
  • Keyboard accessibility for all interactive elements
  • Clear focus indicators with 3px outline for visibility
  • Logical tab order throughout the site
  • Touch targets meet minimum 44px requirement

Content and Media

  • Proper heading hierarchy (H1-H6) for screen reader navigation
  • Descriptive alt text for all images
  • High contrast color ratios (4.5:1 minimum)
  • Responsive design supporting 200% zoom without horizontal scrolling
  • Text resizing support up to 200% without loss of functionality

Forms and Dynamic Content

  • Form labels properly associated with input fields
  • Error messages clearly linked to relevant form fields
  • Real-time validation feedback with screen reader announcements
  • ARIA live regions for dynamic content updates
  • Loading states communicated to assistive technology

Assistive Technology Support

  • Screen reader compatibility (NVDA, JAWS, VoiceOver)
  • Semantic HTML markup for proper content structure
  • ARIA labels and landmarks for enhanced navigation
  • Focus management for modal dialogs and overlays
  • High contrast mode support
  • Reduced motion support for users with vestibular disorders
{% endif %}

Testing and Validation

Our accessibility implementation has been verified through:

  • Automated testing using industry-standard accessibility scanners
  • Manual testing with keyboard-only navigation
  • Screen reader testing with NVDA, JAWS, and VoiceOver
  • Color contrast analysis using professional accessibility tools
  • Mobile accessibility testing across different devices and screen sizes
  • Focus management testing for dynamic content and forms
